# 12_ML_Examples — Sample Monitoring Outputs

## Purpose

Sample output files generated by the ML monitoring pipeline during development and testing runs.

These files are **not committed** to the repository (see `.gitignore`) as they may contain trade-level data. They serve as reference examples for expected output formats.

## File types

| Extension | Description |
|---|---|
| `*.csv` | Sample trade exports or monitoring run outputs |
| `*.log` | Strategy instance log files |
| `*.md` | Health report samples |

## Expected output format from the monitoring pipeline

After running `monitoring_engine.py`, the system produces:

```
08_Monitoring/
├── weekly_monitor_report.csv    → Weekly drift report with traffic light signal
├── monthly_monitor_report.csv   → Monthly performance review
├── daily_metrics.csv            → Daily granularity metrics
├── weekly_metrics.csv           → Weekly aggregated metrics
└── portfolio_summary.json       → Latest portfolio health snapshot
```
